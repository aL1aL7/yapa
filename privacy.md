# Privacy Policy – YAPA

**Last updated:** April 2026

---

## 1. Controller

This app is provided as a private open-source project. Privacy-related enquiries can be submitted via the GitHub repository.

---

## 2. Principle

YAPA is a plain client for self-hosted [Paperless-ngx](https://docs.paperless-ngx.com/) instances. The app collects **no** usage data, contains **no** advertising, uses **no** analytics or tracking services, and transmits **no** data to third parties or to any servers operated by the app developer.

All network communication takes place exclusively between the user's device and the Paperless-ngx server operated by the user themselves.

---

## 3. Data Processed

### 3.1 Credentials

The following data is required to connect to the Paperless-ngx server:

| Data | Purpose | Storage location |
|---|---|---|
| Server URL | Establishing a connection to the user's own instance | Device (Android Keystore) |
| Username / password **or** API token | Authentication against the user's own server | Device (Android Keystore) |
| TLS certificate fingerprint | Detection of certificate changes (TOFU pinning) | Device (Android Keystore) |

Credentials are stored exclusively on-device in the **Android Keystore** (via `flutter_secure_storage`). They are never transmitted to the cloud or shared with other apps.

### 3.2 App Settings

Display preferences (e.g. preferred language, default view, layout options) are stored locally in the app's on-device storage and are never transmitted.

### 3.3 Temporary Files

When uploading a document via the Android share-target feature, the shared file is briefly cached in a temporary app directory. This file is not retained after the upload completes.

Downloaded documents are saved to the location chosen by the user on their device; no further processing by the app takes place.

---

## 4. Permissions

| Permission | Reason |
|---|---|
| **Internet** | Connection to the user's Paperless-ngx server |
| **Camera** | Taking photos for direct document upload |
| **Media / file access** | Selecting existing files from the device for upload |

Camera and file-access permissions are only used in response to an explicit action by the user (taking a photo or selecting a file).

---

## 5. No Sharing with Third Parties

YAPA transmits no data to any third party. No analytics SDKs (e.g. Firebase, Google Analytics), advertising networks, or other external services are integrated.

The only network traffic generated by the app is directed to the Paperless-ngx server whose URL the user configures themselves.

---

## 6. Data Security

- All connections to the Paperless-ngx server use **HTTPS** exclusively. Unencrypted HTTP connections are rejected by the app.
- Credentials are stored in the **Android Keystore**, which is protected by the device's hardware security chip.
- An optional **TOFU certificate pinning** (Trust On First Use) mechanism detects changes to the server certificate and alerts the user.
- Screenshots and screen recordings of the app are blocked at the system level via the Android `FLAG_SECURE` flag.
- Automatic backup of app data by Android (Auto-Backup / Cloud Backup) is disabled.

---

## 7. User Rights

Because YAPA itself does not store or process any personal data, all data associated with the app (credentials, settings) resides solely on the user's device. It can be removed at any time by using **Log out** in the app settings or by fully **uninstalling** the app.

The user is solely responsible for any data stored on their own Paperless-ngx server.

---

## 8. Children

YAPA is not directed at children under the age of 16. The app does not knowingly collect data from children.

---

## 9. Changes to This Privacy Policy

If significant changes are made to the app that affect data processing, this policy will be updated accordingly. The current version is always available in the app's GitHub repository.

---

## 10. Contact

For questions or comments regarding privacy, please contact the developer via the GitHub repository:  
**https://github.com/al1al7/yapa**

# Datenschutzerklärung – YAPA

**Stand:** April 2026

---

## 1. Verantwortlicher

Diese App wird als privates Open-Source-Projekt bereitgestellt. Anfragen zum Datenschutz können über das Repository auf GitHub gestellt werden.

---

## 2. Grundsatz

YAPA ist ein reiner Client für selbst gehostete [Paperless-ngx](https://docs.paperless-ngx.com/)-Instanzen. Die App erhebt **keine** Nutzungsdaten, enthält **keine** Werbung, verwendet **keine** Analyse- oder Tracking-Dienste und übermittelt **keine** Daten an Dritte oder an Server des App-Entwicklers.

Sämtliche Kommunikation findet ausschließlich zwischen dem Gerät des Nutzers und dem vom Nutzer selbst betriebenen Paperless-ngx-Server statt.

---

## 3. Verarbeitete Daten

### 3.1 Zugangsdaten

Zur Anmeldung am Paperless-ngx-Server werden folgende Daten benötigt:

| Datum | Zweck | Speicherort |
|---|---|---|
| Server-URL | Verbindungsaufbau zur eigenen Instanz | Gerät (Android Keystore) |
| Benutzername / Passwort **oder** API-Token | Authentifizierung gegenüber dem eigenen Server | Gerät (Android Keystore) |
| TLS-Zertifikats-Fingerabdruck | Erkennung von Zertifikatsänderungen (TOFU-Pinning) | Gerät (Android Keystore) |

Zugangsdaten werden ausschließlich lokal auf dem Gerät im **Android Keystore** gespeichert (via `flutter_secure_storage`). Sie werden weder in die Cloud übertragen noch mit anderen Apps geteilt.

### 3.2 App-Einstellungen

Anzeigeeinstellungen (z. B. bevorzugte Sprache, Standard-Ansicht, Darstellungsoptionen) werden lokal im App-Speicher des Geräts abgelegt und nicht übertragen.

### 3.3 Temporäre Dateien

Beim Hochladen von Dokumenten über die Teilen-Funktion wird die geteilte Datei kurzzeitig in einem temporären App-Verzeichnis zwischengespeichert. Diese Datei wird nach dem Upload nicht dauerhaft gespeichert.

Heruntergeladene Dokumente werden an dem vom Nutzer gewählten Speicherort auf dem Gerät gespeichert; darüber hinaus erfolgt keine Verarbeitung durch die App.

---

## 4. Berechtigungen

| Berechtigung | Grund |
|---|---|
| **Internet** | Verbindung zum Paperless-ngx-Server des Nutzers |
| **Kamera** | Aufnahme von Fotos für den direkten Dokumenten-Upload |
| **Medienzugriff / Dateizugriff** | Auswahl vorhandener Dateien vom Gerät für den Upload |

Die Kamera- und Dateizugriffs-Berechtigungen werden nur auf ausdrückliche Aktion des Nutzers (Foto aufnehmen, Datei auswählen) aktiv genutzt.

---

## 5. Keine Weitergabe an Dritte

YAPA übermittelt keinerlei Daten an Dritte. Es werden keine Analyse-SDKs (z. B. Firebase, Google Analytics), Werbenetzwerke oder sonstigen externen Dienste eingebunden.

Der einzige Netzwerkverkehr der App führt zum Paperless-ngx-Server, dessen URL der Nutzer selbst konfiguriert.

---

## 6. Datensicherheit

- Alle Verbindungen zum Paperless-ngx-Server erfolgen ausschließlich über **HTTPS**. Unverschlüsselte HTTP-Verbindungen werden von der App abgelehnt.
- Zugangsdaten werden im **Android Keystore** gespeichert, der durch den Hardware-Sicherheitschip des Geräts geschützt ist.
- Ein optionales **TOFU-Zertifikats-Pinning** (Trust On First Use) erkennt Änderungen am Server-Zertifikat und warnt den Nutzer.
- Screenshots und Bildschirmaufnahmen der App werden durch das Android-Flag `FLAG_SECURE` systemseitig unterbunden.
- Ein automatisches Backup der App-Daten durch Android (Auto-Backup / Cloud-Backup) ist deaktiviert.

---

## 7. Rechte der Nutzer

Da YAPA selbst keine personenbezogenen Daten speichert oder verarbeitet, liegen alle im Zusammenhang mit der App gespeicherten Daten (Zugangsdaten, Einstellungen) ausschließlich lokal auf dem Gerät des Nutzers. Sie können jederzeit durch **Abmelden** in den App-Einstellungen oder durch vollständiges **Deinstallieren** der App entfernt werden.

Für Daten, die auf dem eigenen Paperless-ngx-Server gespeichert sind, ist der Nutzer selbst verantwortlich.

---

## 8. Kinder

YAPA richtet sich nicht an Kinder unter 16 Jahren. Die App erhebt wissentlich keine Daten von Kindern.

Bei wesentlichen Änderungen der App, die die Datenverarbeitung betreffen, wird diese Erklärung aktualisiert. Die jeweils aktuelle Version ist im App-Repository auf GitHub einsehbar.
